2.	ОБЩИЕ ПОЛОЖЕНИЯ
2.1.	 Назначение документа
Единый документ, содержащий краткую информацию о всех доступных Бизнес-сервисах с указанием 
реализуемой функциональности, доступных уровней, их параметрах и ответственных со стороны ИТ.  
Данный документ является основным документом для краткого описания всех сервисов 
предоставляемых Исполнителем и соответствует принятому в ________ процессу Управления Уровнем 
Сервисов (Service Level Management, SLM).  

2.2.	 Используемые классификаторы и термины
Иерархия уровней сервисов
«Уровень сервиса» - свойство сервиса, определяемое комбинацией возможных значений различных 
параметров сервиса. Уровень сервиса является предметом договоренностей по сервису и должен 
быть зафиксирован в SLA. 
 
Параметрами, определяющими уровень сервиса, являются:
•	Время предоставления сервиса;
•	Время поддержки;
•	Время реакции на Заявку Пользователя; 
•	Время решения Заявки Пользователя;
Каждый параметр ограничивается определенным набором значений. Комбинации значений различных 
параметров (в зависимости от количества параметров) могут давать иерархию уровней по степени 
их специфичности к конкретному сервису и Заказчику: 
1.	Специфический – комбинация параметров учитывает специфику данного сервиса
2.	Индивидуальный – комбинация специфических параметров для данного сервиса и индивидуальных 
для Заказчика

Подключение сервиса
Предоставление пользователям возможностей использовать функциональность и получать поддержку 
по конкретному ИТ-сервису. 

Отключение сервиса
Завершение предоставления пользователям возможностей использовать функциональность и 
получать поддержку по конкретному ИТ-сервису

Время предоставления
Время, в течение которого пользователи имеют возможность пользоваться функциональностью
данного ИТ-сервиса.

Время поддержки
Время, в течение которого пользователям предоставляется поддержка по данному сервису.

Время реакции по Заявке пользователя
Время от момента регистрации Заявки в Service Desk до начала работ по ней Специалиста команды
поддержки должно быть не более 30 минут. 

Время решения обращений
Время, от момента регистрации Заявки в в Service Desk, до момента установления у Заявки 
статуса «Решено» (т.е. крайний срок решения обращения Пользователя). Время определяется в 
рабочих часах команды поддержки.

Недоступность сервиса
Отсутствие возможности пользоваться функциональностью данного сервиса.

Правила приоритезации инцидентов по влиянию:
При регистрации для обработки каждой Заявки устанавливается следующие параметры:
•	Влияние – отражает степень влияния произошедшего инцидента;
•	Приоритет – отражает важность данной Заявки и однозначно определяет Крайний срок для Заявок категории «Инцидент»;
•	Крайний срок – отражает время, до которого Заявка должна быть решена. 

У параметра «Влияние» возможны следующие значения:
•	Высокое (на Отдел);
•	Среднее (на Группу);
•	Низкое (на одного Сотрудника);

Значения параметра «Приоритет» и «Крайний срок» по данному приоритету для Заявок категории 
«Инцидент»  определены в предыдущем разделе. Значения параметров «Приоритет» и «Крайний срок»
для Заявок категории «Запрос на обслуживание» (ЗНО) определяется индивидуально для каждого 
типа  ЗНО.

Каждый Уровень сервиса содержит взаимосвязь различных значений параметров «Влияние» и 
«Приоритет». Также в Уровне сервиса указывается время работы команды поддержки по данному
сервису. Расчет крайних сроков по решению Заявок пользователей производится с учетом графика
работы команды поддержки по данному сервису. При регистрации Заявки категории «Инцидент» по
сервису, который связан с Заявкой для данного пользователя, автоматически определяется
соответствующий Уровень сервиса. Координатор Service Desk определяет «Влияние» данного
инцидента, «Приоритет» и «Крайний срок» устанавливаются автоматически. 
Для Заявок категории «Запрос на обслуживание» (ЗНО) Координатор Service Desk указывает 
тип ЗНО. Значения параметров «Приоритет» и «Крайний срок» устанавливаются автоматически на
 основе типа ЗНО.
